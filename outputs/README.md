# Output status and authority

The files already present under `outputs/reports`, `outputs/tables`, and
`outputs/figures` are historical development artifacts from several superseded
experiment generations.  They are retained for auditability, not as current
Route-A evidence.  In particular, they may contain old sample sizes, the already
inspected 2019–2020 development-evaluation split described as a “blind test,”
random station holdouts described as transfer, or ecological/regulatory and
calibration language that the current protocol expressly withdraws.  Do not cite
those files as confirmation results or as the current manuscript's claims.

The three legacy identifiers `b1`, `s2`, and `p3` are ordinary monitoring sites.
Their file or display order does not establish a reservoir cascade, hydraulic
connectivity, regulation status, or travel time.

Current Route-A evidence is authoritative only when all of the following agree:

1. a content-addressed development run and its validated Stage-09 receipt;
2. the separate matched-control Stage-09b completion receipt;
3. the frozen model-suite and isolated development-replay receipts;
4. the pre-label chronology, inference, outcome-QC, and authorization gates; and
5. after the single raw-label acquisition, a fully validated canonical
   `outputs/confirmatory/route_a_*/opening_receipt_v1.json` and its bound trusted
   artifacts.

Absent that complete chain, an output is development-only, incomplete, or
historical.  The canonical manuscript scope is defined by the frozen claim ledger
and the byte-verified documents listed there, not by filenames in this directory.
